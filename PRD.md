## 1. 프로젝트 개요

### 1.1 프로젝트 목적
데이터 엔지니어링, 웹 개발, 클라우드 서비스 등 기술 관련 지식을 공유하고 기록하는 개인 기술 블로그입니다. SEO 최적화를 통해 검색 유입을 확보하고, 고품질 콘텐츠 제공을 통해 애드센스 승인 및 수익화를 목표로 합니다.

### 1.2 핵심 가치
- **기술 지식 공유**: 실무 경험을 바탕으로 한 실용적인 기술 가이드 제공
- **SEO 최적화**: 검색 엔진 최적화를 통한 유기적 트래픽 확보
- **사용자 경험**: 반응형 디자인과 직관적인 네비게이션으로 최적의 읽기 경험 제공
- **지속 가능성**: 정기적인 콘텐츠 업데이트를 통한 장기적 운영

## 3. 기능 요구사항

### 3.1 핵심 기능

#### 3.1.1 콘텐츠 관리
- **포스트 작성**: Markdown 형식의 블로그 포스트 작성
- **카테고리 분류**: Python, AWS, ChatGPT, SEO, Airflow, Notion 등 카테고리별 분류
- **태그 시스템**: 세부 주제별 태그를 통한 콘텐츠 분류
- **목차(TOC)**: 긴 글의 가독성을 위한 자동 목차 생성
- **이미지 관리**: GitHub Issues를 활용한 이미지 호스팅 및 최적화

#### 3.1.2 사용자 인터페이스
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 모든 기기 지원
- **사이드바 네비게이션**: 카테고리별 콘텐츠 탐색
- **검색 기능**: 사이트 내 콘텐츠 검색 (현재 미활성화, 향후 구현 예정)
- **RSS 피드**: `/rss/` 경로를 통한 RSS 구독 지원

#### 3.1.3 소셜 기능
- **댓글 시스템**: Disqus를 통한 댓글 기능
- **소셜 공유**: 포스트 공유 기능
- **관련 포스트**: 자동 관련 포스트 추천

#### 3.1.4 SEO 및 분석
- **SEO 최적화**: 메타 태그, 구조화된 데이터, 사이트맵 자동 생성
- **Google Analytics**: 트래픽 분석 및 사용자 행동 추적
- **Search Console**: Google, Bing, Naver 검색 엔진 등록 완료
- **사이트맵**: 자동 생성되는 XML 사이트맵

### 3.2 필수 페이지
- ✅ **홈페이지** (`/`): 최신 포스트 목록
- ✅ **소개 페이지** (`/about/`): 블로그 및 작성자 소개
- ✅ **연락처** (`/contact/`): 문의 및 피드백
- ✅ **개인정보처리방침** (`/privacy/`): 법적 고지
- ✅ **이용약관** (`/terms/`): 법적 고지
- ✅ **검색 페이지** (`/search/`): 사이트 내 검색
- ✅ **카테고리 페이지** (`/categories/`): 카테고리별 포스트 목록

## 6. SEO 전략

### 6.1 기술적 SEO
- ✅ **사이트맵**: 자동 생성 (`/sitemap.xml`)
- ✅ **로봇.txt**: 검색 엔진 크롤링 최적화
- ✅ **구조화된 데이터**: JSON-LD 스키마 마크업
- ✅ **모바일 친화적**: 반응형 디자인
- ✅ **페이지 속도**: 정적 사이트로 빠른 로딩
- ✅ **HTTPS**: SSL 인증서 적용

### 6.2 콘텐츠 SEO
- **제목 최적화**: 30-60자, 타겟 키워드 포함
- **메타 설명**: 120-160자, 클릭 유도 문구
- **헤딩 구조**: H1-H4 계층적 구조
- **내부 링크**: 관련 포스트 간 연결
- **이미지 최적화**: Alt 텍스트, 파일명 키워드 포함
- **롱테일 키워드**: 구체적이고 경쟁이 낮은 키워드 타겟팅

### 6.3 검색 엔진 등록
- ✅ **Google Search Console**: 등록 및 사이트맵 제출 완료
- ✅ **Bing Webmaster Tools**: 등록 완료
- ✅ **Naver Search Advisor**: 등록 완료

## 13. 프로젝트 구조

### 13.1 디렉토리 구조
```
mustarddata/
├── _config.yml          # 사이트 설정
├── _data/               # 데이터 파일 (네비게이션, UI 텍스트)
├── _includes/           # 재사용 가능한 HTML 컴포넌트
├── _layouts/            # 페이지 레이아웃
├── _pages/              # 정적 페이지 (About, Contact 등)
├── _posts/              # 블로그 포스트
├── _sass/               # SCSS 스타일시트
├── assets/              # 이미지, CSS, JS 파일
└── _site/               # 빌드된 정적 사이트 (자동 생성)
```

### 13.2 주요 설정 파일
- `_config.yml`: 사이트 전역 설정
- `_data/navigation.yml`: 네비게이션 메뉴 구조
- `Gemfile`: Ruby 의존성 관리
- `package.json`: Node.js 의존성 관리
