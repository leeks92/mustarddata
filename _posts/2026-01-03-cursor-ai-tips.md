## 들어가며

Cursor AI는 개발자의 코딩 생산성을 혁신적으로 향상시키는 AI 기반 코드 에디터입니다. ChatGPT나 GitHub Copilot과는 다른 접근 방식으로, 코드베이스 전체를 이해하고 맥락에 맞는 코드를 생성해줍니다.

이 글에서는 Cursor AI를 효과적으로 활용하는 실전 팁과 노하우를 공유합니다.

## Cursor AI란?

### Cursor AI의 특징

Cursor AI는 다음과 같은 특징을 가지고 있습니다:

1. **코드베이스 전체 이해**: 프로젝트의 모든 파일을 분석하여 맥락을 파악
2. **자동 코드 생성**: 자연어 설명만으로 코드를 자동 생성
3. **리팩토링 지원**: 기존 코드를 개선하고 최적화
4. **버그 수정**: 에러를 찾아내고 해결책 제시
5. **문서화 자동화**: 코드 주석과 문서 자동 생성

### Cursor AI vs 다른 도구

| 기능 | Cursor AI | GitHub Copilot | ChatGPT |
|------|-----------|---------------|---------|
| 코드베이스 이해 | ✅ 전체 분석 | ❌ 제한적 | ❌ 없음 |
| 실시간 제안 | ✅ | ✅ | ❌ |
| 대화형 인터페이스 | ✅ | ❌ | ✅ |
| 파일 간 맥락 이해 | ✅ | ⚠️ 제한적 | ❌ |

## 기본 사용법

### 1. Chat 기능 활용하기

Cursor의 Chat 기능은 코드베이스 전체를 이해한 상태에서 대화할 수 있습니다.

**효과적인 사용 예시:**

```
❌ 나쁜 예: "함수 만들어줘"
✅ 좋은 예: "사용자 인증을 위한 JWT 토큰 검증 함수를 만들어줘. 
           기존 auth.js 파일의 구조를 참고해서 작성해줘."
```

**프롬프트 작성 팁:**

1. **구체적으로 작성**: "함수 만들어줘"보다 "사용자 로그인을 처리하는 async 함수를 만들어줘"가 더 좋습니다.
2. **맥락 제공**: 관련 파일이나 기존 코드 스타일을 언급하세요.
3. **요구사항 명시**: 에러 처리, 타입 정의, 테스트 코드 등 필요한 것들을 명시하세요.

### 2. Composer 기능으로 대규모 변경

Composer는 여러 파일을 동시에 수정할 수 있는 강력한 기능입니다.

**사용 시나리오:**

- 새로운 기능 추가 (여러 파일에 걸친 변경)
- 리팩토링 (전체 코드베이스 구조 변경)
- 마이그레이션 (프레임워크 업그레이드 등)

**예시 프롬프트:**

```
"React 컴포넌트를 Vue 3 Composition API로 마이그레이션해줘. 
기존 props와 상태 관리를 모두 유지하면서 변환해줘."
```

### 3. Inline Edit로 빠른 수정

코드 블록을 선택하고 `Cmd/Ctrl + K`를 눌러 인라인으로 수정할 수 있습니다.

**사용 팁:**

```javascript
// 코드 선택 후 Cmd+K
// 프롬프트: "에러 처리를 추가하고 로깅을 개선해줘"

// Before
function fetchData(url) {
  return fetch(url).then(res => res.json());
}

// After (AI가 생성)
async function fetchData(url) {
  try {
    const response = await fetch(url);
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    const data = await response.json();
    console.log('Data fetched successfully:', data);
    return data;
  } catch (error) {
    console.error('Error fetching data:', error);
    throw error;
  }
}
```

## 고급 활용 팁

### 1. 컨텍스트 제공하기

Cursor AI는 더 많은 컨텍스트를 제공할수록 더 정확한 코드를 생성합니다.

**효과적인 컨텍스트 제공 방법:**

```
"기존 UserService 클래스의 패턴을 따라서 
ProductService 클래스를 만들어줘. 
동일한 에러 처리 방식과 로깅 스타일을 사용해줘."
```

**@ 기호 활용:**

- `@파일명`: 특정 파일을 컨텍스트에 포함
- `@폴더명`: 폴더 내 모든 파일 포함
- `@코드`: 코드 블록을 직접 참조

```
"@auth.js 파일의 구조를 참고해서 
@middleware 폴더에 인증 미들웨어를 만들어줘"
```

### 2. 단계별 요청하기

복잡한 작업은 작은 단계로 나누어 요청하는 것이 효과적입니다.

**예시:**

```
1단계: "데이터베이스 스키마를 설계해줘"
2단계: "위 스키마를 기반으로 모델 클래스를 만들어줘"
3단계: "모델을 사용하는 서비스 레이어를 구현해줘"
4단계: "서비스를 사용하는 API 엔드포인트를 만들어줘"
```

### 3. 코드 스타일 일관성 유지

프로젝트의 기존 코드 스타일을 학습하도록 명시하세요.

```
"기존 코드 스타일을 따라서 작성해줘. 
함수형 프로그래밍 패러다임을 사용하고, 
타입스크립트 strict 모드를 준수해줘."
```

### 4. 테스트 코드 함께 생성

코드와 함께 테스트 코드도 생성하도록 요청하세요.

```
"사용자 인증 함수를 만들어주고, 
Jest를 사용한 단위 테스트도 함께 작성해줘."
```

## 실전 활용 시나리오

### 시나리오 1: 새로운 기능 추가

**상황**: 사용자 프로필 페이지에 아바타 업로드 기능 추가

**프롬프트:**

```
"사용자 프로필 페이지에 아바타 이미지 업로드 기능을 추가해줘.

요구사항:
1. 프론트엔드: React 컴포넌트로 이미지 업로드 UI
2. 백엔드: Express API 엔드포인트 (multer 사용)
3. 이미지 리사이징 및 최적화
4. 기존 User 모델에 avatar 필드 추가
5. 에러 처리 및 유효성 검사 포함

기존 코드 스타일을 유지하면서 작성해줘."
```

### 시나리오 2: 버그 수정

**상황**: API 응답이 느린 문제 해결

**프롬프트:**

```
"@api.js 파일의 성능 문제를 분석하고 최적화해줘.

문제:
- API 응답 시간이 3초 이상 걸림
- 데이터베이스 쿼리가 비효율적일 가능성

해결책:
1. N+1 쿼리 문제 해결
2. 필요한 경우 캐싱 추가
3. 응답 시간을 500ms 이하로 개선"
```

### 시나리오 3: 리팩토링

**상황**: 클래스 기반 컴포넌트를 함수형으로 변환

**프롬프트:**

```
"@components 폴더의 모든 클래스 컴포넌트를 
React Hooks를 사용한 함수형 컴포넌트로 변환해줘.

요구사항:
1. 모든 상태를 useState로 변환
2. 생명주기 메서드를 useEffect로 변환
3. 성능 최적화를 위해 useMemo, useCallback 활용
4. 기존 기능과 동작은 완전히 동일하게 유지"
```

### 시나리오 4: 문서화

**상황**: API 문서 자동 생성

**프롬프트:**

```
"@routes/api.js 파일의 모든 엔드포인트에 대해 
JSDoc 형식의 주석을 추가해줘.

각 엔드포인트마다:
- 설명
- 파라미터 타입 및 설명
- 응답 형식
- 에러 케이스
- 사용 예시

를 포함해줘."
```

## 프롬프트 작성 베스트 프랙티스

### ✅ DO (해야 할 것)

1. **구체적으로 작성**
   ```
   ✅ "사용자 로그인을 처리하는 async 함수를 만들어줘. 
      이메일과 비밀번호를 받아서 JWT 토큰을 반환해줘."
   
   ❌ "로그인 함수 만들어줘"
   ```

2. **맥락 제공**
   ```
   ✅ "@auth.js 파일의 패턴을 따라서 
      새로운 인증 미들웨어를 만들어줘"
   
   ❌ "인증 미들웨어 만들어줘"
   ```

3. **요구사항 명시**
   ```
   ✅ "에러 처리, 로깅, 타입 정의를 포함해서 작성해줘"
   
   ❌ "함수 만들어줘"
   ```

4. **단계별 요청**
   ```
   ✅ "1단계: 스키마 설계, 2단계: 모델 생성, 3단계: API 구현"
   
   ❌ "전체 기능을 한 번에 만들어줘"
   ```

### ❌ DON'T (하지 말아야 할 것)

1. **너무 모호한 요청**
   ```
   ❌ "좋은 코드 만들어줘"
   ❌ "최적화해줘" (어떤 부분을 어떻게 최적화할지 명시)
   ```

2. **컨텍스트 없이 요청**
   ```
   ❌ "함수 만들어줘" (어떤 함수인지, 어디에 사용할지 불명확)
   ```

3. **한 번에 너무 많은 요청**
   ```
   ❌ "전체 애플리케이션을 처음부터 만들어줘"
   ✅ 작은 단위로 나누어 요청
   ```

## Cursor AI 설정 최적화

### 1. 모델 선택

Cursor는 여러 AI 모델을 지원합니다:

- **Claude 3.5 Sonnet**: 복잡한 추론과 대규모 변경에 적합
- **GPT-4**: 빠른 응답과 일반적인 코딩 작업에 적합
- **GPT-3.5**: 빠르지만 정확도가 낮음

**권장 설정:**
- 일반 코딩: GPT-4
- 복잡한 리팩토링: Claude 3.5 Sonnet
- 빠른 수정: GPT-3.5

### 2. 컨텍스트 윈도우 설정

더 많은 파일을 참조하려면 컨텍스트 윈도우를 늘리세요.

**설정 방법:**
```
Settings → Features → Codebase Indexing
→ Increase context window size
```

### 3. 자동 완성 설정

**권장 설정:**
- **Inline suggestions**: 활성화
- **Trigger on**: 타이핑 중 자동 제안
- **Accept suggestion**: Tab 키

## 주의사항 및 한계

### 1. 생성된 코드 검토 필수

AI가 생성한 코드는 항상 검토하고 테스트해야 합니다.

```javascript
// AI가 생성한 코드 예시
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0);
}

// 검토 필요 사항:
// - 타입 체크
// - 에러 처리
// - 엣지 케이스
// - 성능 최적화
```

### 2. 보안 고려사항

AI가 생성한 코드에 보안 취약점이 있을 수 있습니다.

**주의해야 할 점:**
- SQL Injection 방지
- XSS 공격 방지
- 인증/인가 로직 검증
- 민감한 정보 하드코딩 방지

### 3. 라이선스 확인

AI가 생성한 코드가 오픈소스 라이선스를 위반할 수 있습니다.

**해결 방법:**
- 생성된 코드의 출처 확인
- 라이선스 호환성 검토
- 필요시 재작성

## 실전 팁 모음

### 팁 1: 코드 리뷰 요청

```
"위에서 생성한 코드를 리뷰하고 개선점을 제시해줘. 
성능, 보안, 가독성 관점에서 분석해줘."
```

### 팁 2: 주석 추가 요청

```
"위 함수에 JSDoc 형식의 주석을 추가해줘. 
파라미터, 반환값, 사용 예시를 포함해줘."
```

### 팁 3: 테스트 케이스 생성

```
"위 함수에 대한 Jest 단위 테스트를 작성해줘. 
정상 케이스와 에러 케이스를 모두 포함해줘."
```

### 팁 4: 성능 최적화

```
"위 코드의 성능을 분석하고 최적화해줘. 
불필요한 리렌더링이나 중복 계산을 제거해줘."
```

### 팁 5: 타입 안정성 강화

```
"위 코드에 TypeScript 타입을 추가해줘. 
strict 모드를 준수하고 any 타입은 사용하지 말아줘."
```

## 마무리

Cursor AI는 강력한 도구이지만, 효과적으로 사용하려면:

1. ✅ **구체적인 프롬프트 작성**: 명확한 요구사항 제시
2. ✅ **맥락 제공**: 관련 파일과 기존 코드 스타일 참조
3. ✅ **단계별 접근**: 복잡한 작업은 작게 나누기
4. ✅ **코드 검토**: 생성된 코드는 항상 검토하고 테스트
5. ✅ **지속적 학습**: 더 나은 프롬프트를 위해 계속 개선

올바르게 활용하면 개발 생산성을 크게 향상시킬 수 있습니다. 이 팁들을 실무에 적용해보시고, 자신만의 노하우를 쌓아가시기 바랍니다.
